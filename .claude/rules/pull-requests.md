# Pull Requests

- PR **titles and descriptions MUST be written in English**, regardless of the language used in the conversation, commits, or codebase comments.
- Keep the conventional-commit style for the title (e.g. `feat(scope): ...`, `fix(scope): ...`).
- Body structure: a short **Summary**, the key **Changes**, and **Testing** notes. Concise over exhaustive.
- This applies to every PR created via `gh`, the `git-create-pr` skill, or any other path.
